MODELO DE DADOS — Agenda dos Grupos CMV-BH
1. Princípio fundamental do modelo
O evento é o núcleo.
Tudo o resto (agenda, e-book, relatórios) nasce dele.
O modelo foi pensado para ser:
* ✔ simples no início
* ✔ extensível no futuro
* ✔ controlado pela PASCOM (administradores)
2. Conceito pastoral do “Evento”
Um Evento representa qualquer atividade da vida comunitária:
Tipos oficiais iniciais
* encontro_grupo
* celebracao
* formacao
* atividade_missionaria
⚠️ Importante
Os administradores poderão acrescentar novos tipos no futuro
(ex.: assembleia, convivência, reunião ampliada, etc.).
3. Estrutura lógica do Evento
3.1 Campos essenciais (núcleo)
Esses sempre existirão:
* id → identificador único
* titulo → nome do evento
* tipo → tipo pastoral
* data_inicio
* data_fim (opcional)
* local
* tema (opcional)
3.2 Organização pastoral
* categoria_principal
	* grupo
	* retiro
* grupo_relacionado (quando aplicável)
	* nome do grupo
	* ou código interno
3.3 Informações descritivas (editáveis)
* descricao
* observacoes
Esses campos são:
* totalmente editáveis
* opcionais
* ideais para enriquecer o e-book
3.4 Visibilidade e publicação
* visibilidade
	* publico
	* interno
Isso controla:
* o que aparece para fora
* o que fica apenas para a comunidade
3.5 Metadados administrativos (PASCOM)
* responsaveis
* criado_por
* data_criacao
* ultima_atualizacao
3.6 Campos personalizados (chave da flexibilidade)
I Aqui est o ponto mais importante do que voc pediu.
"campos_personalizados": {
  "pregador": "Pe. João",
  "material": "link-do-drive",
  "cor_liturgica": "verde"
}

✔ Administradores podem:
* criar novos campos
* remover campos
* adaptar linguagem pastoral
* sem alterar o código do sistema
